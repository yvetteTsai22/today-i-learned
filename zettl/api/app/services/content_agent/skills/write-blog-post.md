# Blog Post Writer

## When to Use
Use this skill when asked to generate content in the "blog" format.

## Tools to Use
- `search_knowledge`: Find additional context related to the topic if the provided source material is insufficient.

## Process
1. Review the provided source material and topic carefully
2. If the source material lacks depth, use `search_knowledge` to find more context
3. Write the post following the structure and requirements below

## Structure
- **Hook**: Open with a compelling question, surprising fact, or relatable scenario
- **Context**: Set the stage — why this topic matters now
- **Insight**: The core idea, explained clearly with depth
- **Examples**: Concrete examples from the source material
- **Takeaway**: Clear, actionable conclusion or call to action

## Requirements
- Length: approximately 1000 words
- Tone: Educational, personal voice — as if explaining to a smart friend
- Include specific examples and details from the source material
- Use subheadings (H2) for each section
- End with a "Key Takeaway" section
- Format as markdown
